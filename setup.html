STEP 01

Copy the Google Sheet template

Required

This Sheet is your card database — every Character, Weapon, Armoury item, Special Order, and Keyword definition the app reads lives here. It's free, and making your own copy means you fully own and control it.

  1. Open the template using the link below.
  2. Go to File → Make a copy, and save it to your own Google Drive.
  3. Rename it to whatever you like — the name doesn't matter to the app.

STEP 02

Deploy the Apps Script

Required

Apps Script is a small, free Google service that lets the app read your Sheet over the internet. You'll paste in a short piece of code, then deploy it as a private web app under your own Google account.

  1. In your copied Sheet, go to Extensions → Apps Script.
  2. Delete any starter code in the editor, and paste in the script provided in your purchase files (Code.gs).
  3. Click Deploy → New deployment.
  4. For "Select type," choose Web app.
  5. Set Execute as to "Me," and Who has access to "Anyone." (This doesn't expose your Sheet data publicly in a meaningful way — it just allows your own app to fetch it. See the FAQ for details.)
  6. Click Deploy, and authorize the script when Google prompts you.
  7. Copy the Web app URL it gives you — it'll end in /exec. You'll need this in Step 5.

STEP 04

Host it on GitHub Pages

Required

GitHub Pages hosts static sites for free, with no traffic limits that matter at this scale. This makes your copy of the app reachable at a URL you can open from any device.

  1. Create a free account at github.com if you don't have one.
  2. Click the + in the top right → New repository. Name it anything (e.g. my-flashpoint-list), and set it to Public.
  3. Once created, click Add file → Upload files, and upload the index.html file from Step 3.
  4. Go to the repository's Settings → Pages.
  5. Under "Source," select Deploy from a branch, choose the main branch and / (root) folder, then click Save.
  6. Wait a minute or two, then refresh — GitHub will show your live URL at the top of that page (something like https://yourname.github.io/my-flashpoint-list/).

STEP 05

First-run setup wizard

Required

Open your new live URL from Step 4. Since this is a fresh copy, the app will show a setup screen instead of jumping straight in.

  1. Paste the Apps Script Web app URL you copied in Step 2 into the first field.
  2. Click Test Connection. You should see a confirmation showing how many rows it found in your Sheet.
  3. Click Save & Continue.

That's it — the app is now fully connected to your Sheet. Anything you add or edit in the Sheet will appear in the app after a refresh.

STEP 06

Connect Supabase for cloud saves

Optional

This step lets you save rosters to the cloud and load them on any device. If you skip it, you can still save and load rosters locally as JSON files — that always works with zero setup.

  1. Create a free account at supabase.com.
  2. Create a new project (pick any name and a database password — store the password somewhere safe, you won't need it for this setup but Supabase requires it).
  3. Once the project is ready, go to the SQL Editor and run the table-creation script provided in your purchase files (supabase-setup.sql) to create the rosters table.
  4. Go to Project Settings → API. Copy the Project URL and the anon public key.
  5. Back in the app, open Settings → Edit Connections, paste both values into the Supabase fields, and click Test Connection.

STEP 07

Run a Health Check

Recommended

Before you start building lists, open Settings → Run Health Check. It scans your Sheet data for the most common setup mistakes — missing fields, duplicate names, keywords with broken links — and tells you exactly what to fix, if anything.

A clean Health Check result means your Sheet is properly structured and you're ready to build your first roster.
